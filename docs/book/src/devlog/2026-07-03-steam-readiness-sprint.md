# Steam Readiness Sprint

Date: 2026-07-03 Tags: review, bugfix, input, scenes, settings, font, pause

## What Changed

Picked the project back up with a Steam release as the goal. A full code
review of the engine surfaced four critical bugs and a long list of shipping
gaps; this sprint fixed the bugs and knocked out the first three foundation
items: executable-relative asset paths, persisted user settings, bitmap font
text rendering, and a pause menu.

## The Review

A subsystem-by-subsystem audit (core loop, rendering, scenes, all ECS systems,
patterns, build/packaging) rated the engine core strong — the fixed-timestep
clock, EnTT discipline, and defensive JSON loading all held up — but found
four critical defects and a set of Steam blockers: no audio, no text
rendering, no persistence, no pause, no Windows build story, CWD-relative
asset paths, and all-placeholder art. The fixes below tracked that list.

## Critical Fixes

**Scene transition use-after-free** — `SceneManager::swap()` destroyed the
scene that called it, mid-`update()`. Transitions are now queued and applied
after the update returns ([ADR-0015](../decisions/0015-deferred-scene-transitions.md)).
Shutdown ordering was also wrong: scenes (and their SDL textures) now tear
down before the renderer, and the gamepad closes before `SDL_Quit()`.

**Input edges dropped on >120 Hz displays** — press edges were computed per
render frame, but a frame can run zero fixed ticks (about half of them on a
240 Hz display), silently eating dashes and melee presses; multi-tick frames
had the opposite bug, replaying one press into several ticks (confirm skipped
straight through the title screen at low fps). Edges now latch until exactly
one fixed tick consumes them (see
[Input Handling](../architecture/input.md#edge-detection-and-latching)).

**Piercing bullets multiplied damage** — no per-target hit tracking meant a
piercing bullet re-applied damage every tick it overlapped an enemy (4–9x
intended). `Piercing` now carries a hit list; each target is damaged once.

**Pattern JSON could melt the game** — `fire_rate: 0` in a pattern file made
an emitter fire a full burst every tick (tens of thousands of entities within
seconds). Numeric fields are now clamped with warnings at load.

Plus a round of moderate fixes: dash no longer truncates post-hit i-frames,
charged shots respect their fire-rate cooldown, freshly spawned enemies
pre-charge their emitters instead of firing the instant they appear, stale
`GameState` no longer leaks score across runs on the victory path, and the
title-screen prompt — drawn at y=300 on a 270-pixel-tall canvas since the day
it was written — is visible for the first time.

## Platform Foundations

**Asset paths** ([ADR-0016](../decisions/0016-executable-relative-asset-paths.md)):
every load goes through `paths::asset()`, anchored to `SDL_GetBasePath()`.
Launching from any working directory now works; the install layout puts
assets next to the binary to match.

**Settings** ([ADR-0017](../decisions/0017-settings-pref-path.md)):
`settings.json` in the SDL pref path carries window scale, fullscreen, vsync,
and volumes — loaded, validated, written back on first run, and actually
applied by the renderer. When a driver refuses vsync, the loop falls back to
a 240 fps frame limiter instead of busy-spinning. `config.json` is read-only
game data again.

## Text Rendering

[ADR-0018](../decisions/0018-bitmap-font-text.md): a monospace bitmap font
atlas (hand-defined 5x7 glyphs, generated by `tools/gen_font.py`) rendered by
`BitmapFont` with per-call colour tinting and integer scaling. The title
screen, character select, game over screen, and HUD score all use real text
now — the era of brightness-encoded score digits is over.

## Pause Menu

Esc/Start pushes `PauseScene` as an overlay: gameplay freezes (only the top
scene updates) and keeps rendering underneath a translucent dim layer, with
Resume / Quit to Title navigation. Two details worth remembering: the scene
below snaps interpolation alpha to 1.0 while covered (a varying alpha makes
frozen sprites shimmer between tick positions), and the overlay's render
deliberately skips `SDL_RenderClear`.

## Build Notes

- SDL >= 3.4 is now enforced in CMake (`SDL_SCALEMODE_PIXELART` needs it)
- The CPM bootstrapper verifies its download hash and skips re-downloading
  when cached — configure works offline / behind restrictive proxies
- `imgui.ini` and `.claude/settings.local.json` are untracked and gitignored
